# Release Notes: v1.0.0

## Summary

`v1.0.0` is the first curated public release of the AppSec Core research artifact set.

This release is designed for:

- academic citation
- paper support
- reviewer inspection
- stable GitHub release archival in Zenodo

This release is intentionally limited to the artifact surface that supports the current `V1` paper set. It is **not** a bundle of later `V2` work, later pilot waves, or broader internal program outputs.

This release is a **research program release**:

- one curated repository snapshot
- one Zenodo-backed release DOI
- a three-paper core release with additional frozen design mirrors for companion study artifacts

## Included Scope

### Papers

- `appsec_core_normalized_ontology.pdf`
- `coverage_preserving_knowledge_compilation.pdf`
- `ontology_grounded_retrieval.pdf`

### Additional frozen design mirrors

- `papers/04-empirical-evaluation/` — frozen OSF-authoritative design mirror
- `papers/05-mcp-se-engineering/` — frozen companion apparatus mirror

### Paper 1 artifacts

- AppSec Core `v0` ontology
- schema and cross-slice vocabulary
- slice contracts

### Paper 2 artifacts

- selected pilot manifests for the first-wave `V1` pilot set:
  - `SSDF`
  - `ASVS`
  - `SLSA`
  - `CIS Controls`
  - `CAPEC Supply Chain`
- selected normalization outputs
- selected gap and comparison outputs directly supporting the released `V1` paper text

### Paper 3 artifacts

- retrieval contract
- minimal runtime snapshot
- worked examples, only if explicitly curated for this release
- verification examples, only if explicitly curated for this release

## Excluded Scope

This release excludes:

- internal working notes
- `V2` prompt packs and delta notes
- raw source captures from framework websites and PDFs
- internal development scripts and experiments
- temporary or contradictory drafts
- environment-specific files and credentials
- the internal development repository as a whole
- later pilot waves and regulatory validation material not required by the current `V1` paper set

## Publication Position

This release should be understood as:

- a curated research publication surface
- not a mirror of internal project history
- not a general-purpose software distribution
- not a partial publication of future `V2` work
- one single public repository release, not a multi-repository paper split

## Recommended Citation Practice

Cite both:

1. the specific paper used
2. the versioned Zenodo-backed repository release
