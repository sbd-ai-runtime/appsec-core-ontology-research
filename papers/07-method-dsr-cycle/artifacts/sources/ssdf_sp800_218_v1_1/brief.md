# SSDF Pilot Brief

## Pilot

- corpus: `SSDF v1.1`
- publication: `NIST SP 800-218`
- publisher: `NIST`

## Goal

Prove that an external AppSec corpus can be:

1. indexed as a stable source artifact
2. normalized into `AppSec Core`
3. compared against the current SbD-ToE manual projection
4. turned into editorial candidates for traceability repair and manual
   completion

## Why SSDF First

- stable official source
- widely recognizable AppSec corpus
- more direct objective/practice/task shape than maturity-heavy corpora
- good first test for `AppSec Core` normalization without starting from a
  regulatory overlay

## Expected Outputs

- source inventory
- normalized external objects
- manual coverage view
- gap analysis summary
- traceability publication candidates

## Non-Goal

This pilot does not update the manual automatically.
