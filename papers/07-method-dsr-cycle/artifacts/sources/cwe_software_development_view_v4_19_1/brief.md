# CWE Pilot Brief

## Pilot

- corpus: `CWE Software Development View`
- publication: `CWE View 699`
- publisher: `MITRE`

## Goal

Prove that a weakness-oriented external corpus can be:

1. captured as a stable official source surface
2. normalized into `AppSec Core`
3. compared against the current SbD-ToE manual projection
4. used to distinguish `manual_gap`, `appsec_core_gap`, `comparison_gap`, and `out_of_scope_for_appsec_core`

## Why CWE Now

- official stable source with machine-readable exports
- good complement to `CAPEC` because it is weakness-oriented, not attack-pattern oriented
- strong pressure on `secure coding`, `input validation`, `unsafe defaults`, and `configuration integrity`
- good next test after `MP-01` to `MP-03` and the `MP-05` backlog decision

## Scope

This pilot starts narrowly with the official `Software Development` view (`699`), not the full CWE universe.

## Expected Outputs

- source inventory
- normalized weakness objects
- manual coverage view
- gap analysis summary
- traceability or method-backlog candidates

## Non-Goal

This pilot does not claim that every CWE entry is in scope for the current `AppSec Core`.
