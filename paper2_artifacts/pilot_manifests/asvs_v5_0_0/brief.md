# ASVS Pilot Brief

## Pilot

- corpus: `OWASP ASVS v5.0.0`
- publication: `OWASP Application Security Verification Standard`
- publisher: `OWASP Foundation`

## Goal

Prove that a requirements-heavy AppSec corpus can be:

1. captured as a stable published source artifact
2. extracted into source objects and units deterministically
3. normalized into `AppSec Core`
4. compared conservatively against the current SbD-ToE manual traceability surface

## Why ASVS Now

- stable official release artifacts exist in machine-readable form
- the corpus is requirement-oriented rather than maturity-only
- it should pressure `requirements`, `validation`, `authentication`, `authorization`, `logging`, and `configuration` surfaces strongly
- it is a better next top-down pilot than `CIS Controls` because it stays closer to application-security semantics

## Expected Outputs

- source inventory
- source unit inventory
- source object inventory
- normalized external objects
- manual coverage view
- traceability comparison read

## Non-Goal

This pilot does not update the manual automatically.
